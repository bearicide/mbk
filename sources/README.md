# MBK Sources

Source/reference area for MBK so playable code stays clean while creative material has a place to land.

## Current source pack

This repo source area is based on the MBK Asset Source Package summary:

- Audio source ideas
- Visual source ideas
- MBK asset structure
- Starter sound philosophy
- Organized folders
- DOCX reference guide

## Repo use

- Keep raw inspiration, references, and notes here.
- Keep production-ready app assets in `/assets`.
- Keep the live app focused on mobile-first recording, quantized loops, and fast pad triggering.
- Do not let source notes accidentally become loaded runtime assets.

## Working folders

Recommended structure:

```text
sources/
  README.md
  audio-sources.md
  visual-sources.md
  source-manifest.md
assets/
  audio/
  images/
  ui/
```

## MBK sound direction

MBK should feel immediate, playable, and rough-edged in the right way: phone-friendly recording, tight quantized loop starts, choke behavior that prevents loop pileups, and starter sounds that help the user play immediately without burying them in sample-pack nonsense.
